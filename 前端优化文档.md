# 前端配色与风格微调方案（仅视觉，不改功能）

版本：v2.1  
日期：2025-01-23  
范围声明：**不改动任何前端/后端功能、组件结构、交互逻辑、提示文案与位置；仅更新色彩、字体、阴影、背景等视觉风格。**

---

## 1. 设计目标与边界
- 保留现有信息架构、页面布局、组件位置与交互流程（按钮、卡片、表单、步骤条、提示文案一律不变）。
- 用“大学生青春活力”风格替换原商务蓝淡风格：更明快的渐变、柔和圆角、轻量阴影、适度贴纸/emoji 点缀。
- 响应式和可访问性保持现状；仅调整视觉变量，不新增功能。

## 2. 配色与字体（建议以 CSS 变量落地）
```css
:root {
  --primary: #6c5ce7;                /* 葡萄紫，主色 */
  --accent: #ff7e67;                 /* 橘珊瑚，强调 */
  --mint: #4ee1c1;                   /* 薄荷绿，成功/正向 */
  --lemon: #ffd166;                  /* 柠檬黄，提示/等待 */
  --ink: #1f2d3d;                    /* 深墨正文 */
  --muted: #8fa3b5;                  /* 次级文字 */
  --bg-gradient: linear-gradient(135deg, #a6c1ff 0%, #fbc2eb 100%);
  --card-surface: #ffffffcc;         /* 玻璃态白 */
  --shadow: 0 10px 30px rgba(48,55,66,0.12);
  --radius-lg: 20px;
  --radius-sm: 12px;
}
```
- 字体：中文 `Smiley Sans` 或 `霞鹜文楷`，英文字体 `Sora`，回退 `-apple-system, "PingFang SC", "Microsoft YaHei", sans-serif`。
- 圆角：按钮/卡片 12-20px；输入框 10-12px；胶囊 999px。
- 阴影：柔和低对比，hover 时适度加强；尊重 `prefers-reduced-motion` 降级为无位移。

## 3. 样式调整要点（保持组件与文案不变）
- **按钮**：主按钮改用 `--primary` 渐变或纯色，hover 上浮 2-3px；禁用态降低饱和度，不改尺寸/文案/位置。
- **卡片**：背景 `--card-surface` + 细描边（1px 透明白）+ 阴影；圆角 16-20px；hover 轻微上浮，无结构变动。
- **标签/步骤条**：用胶囊色块（primary/mint/lemon），字体加粗；步骤圆点颜色替换即可，不改步骤数量与文案。
- **表单/输入框**：浅灰底 `#f7f9fc`，聚焦描边用 `--mint`；占位符颜色稍淡；不改尺寸与布局。
- **背景**：全局或首屏使用 `--bg-gradient`；局部容器可用玻璃态白；不添加/移除模块。
- **提示/空态/Toast**：颜色替换为新主色/强调色，图标可用轻松的 emoji，但保持文案与出现位置不变。

## 4. 需修改的文件（仅视觉层）
- 样式：`static/css/design.css`（全局/组件色板）、`static/css/campus-animations.css`（若保留，可仅调色）、`static/css/campus-mobile.css`（必要时同步变量）。  
- 模板引用：保持现有 `<link>` 与 JS 引用，不新增/删除组件结构。  
- 不新增额外 JS 功能；仅可保留/调整现有动画的颜色与时长（若需要降噪）。

## 5. 落地步骤（不动结构）
1) **引入变量**：在主样式文件定义上方新增/替换上述变量（或统一抽到一个变量段），不改选择器结构。  
2) **替换色彩**：用变量替换硬编码颜色（按钮、卡片、标签、步骤条、输入框、Toast、空态等），保持 class 与 DOM 不变。  
3) **背景与阴影**：页面背景改为 `--bg-gradient` 或浅灰，卡片/弹窗使用玻璃态 + 阴影；不移动任何元素。  
4) **hover/active**：统一 hover 上浮 2-3px、阴影加强；移动端禁用 hover 位移。  
5) **动效降级**：尊重 `prefers-reduced-motion`；动画总时长 ≤600ms；不新增复杂动效。  
6) **移动端检查**：375/414/768 宽度下仅验证颜色与可读性，不调布局。

## 6. 验收清单
- 页面布局、组件位置、按钮/表单/步骤文案与逻辑完全未动，仅颜色/阴影/背景更新。  
- 主色/强调/成功/提示色均替换为新变量；无遗留旧商务蓝。  
- 卡片、按钮、标签圆角与阴影已统一；禁用态可见。  
- 背景与容器风格统一（渐变或玻璃态），无新增/删除模块。  
- 动效/hover 正常，`prefers-reduced-motion` 生效；移动端颜色对比度良好。  
- 构建/运行无需额外依赖，HTML/JS 功能保持原样。

---

说明：本方案仅指导“换皮肤”式视觉微调，严禁改动现有功能和交互流程。如需更换插画/贴纸，请放在 `static/images` 对应目录并保持现有引用接口。保持所有模板结构和 JS 逻辑不变。***
