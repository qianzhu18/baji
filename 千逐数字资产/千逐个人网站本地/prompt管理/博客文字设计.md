# 千逐个人网站 - 整体设计方案

## 项目概述

基于命令行界面（CLI）风格，打造一个融合中国古典美学与现代技术的个人展示网站。项目参考 [inhai.wiki](https://inhai.wiki/) 的设计理念，结合千逐的个人特色，创建一个独特的交互式个人主页。

## 设计理念

### 核心主题
- **千千君子，温润如玉** - 体现东方君子品格的温润内敛
- **命令行美学** - 技术感与极简主义的完美结合
- **第一性原理** - 简洁直接的信息架构
- **反脆弱性** - 适应性强的响应式设计

### 视觉风格
- **色彩体系**：以玉石色调为主
  - 纸白色（Paper White）: `#f5f5f1` - 主背景
  - 墨绿色（Ink Green）: `#3a4f4b` - 主文字
  - 青玉蓝（Jade Blue）: `#4a7c59` - 链接和强调
  - 印章红（Seal Red）: `#c0504d` - 高亮和交互

- **字体系统**：
  - 英文：Geist Sans/Mono - 现代简洁
  - 中文：Noto Serif SC - 传统雅致
  - 代码：等宽字体保持技术感

## 技术架构

### 前端技术栈
- **框架**：Next.js 14 (App Router)
- **样式**：Tailwind CSS 4.1.9 + 自定义玉石主题
- **UI组件**：Radix UI + shadcn/ui
- **字体**：Geist + Noto Serif SC
- **动画**：Tailwind Animate + 自定义CSS动画
- **部署**：Vercel
- **分析**：Vercel Analytics

## 开发与部署流程

### Git版本控制
```bash
# 初始化仓库
git init
git remote add origin <github-repo-url>

# 分支策略
main          # 生产环境分支
develop       # 开发分支
feature/*     # 功能分支
hotfix/*      # 紧急修复分支
release/*     # 发布准备分支
```

### 开发工作流
1. **环境准备**
   ```bash
   cd 前端初始
   pnpm install
   cp .env.example .env.local  # 配置环境变量
   pnpm dev
   ```

2. **功能开发**
   ```bash
   git checkout -b feature/cli-commands-enhancement
   # 开发新功能
   git add .
   git commit -m "feat: add enhanced CLI commands system"
   git push origin feature/cli-commands-enhancement
   ```

3. **代码质量保证**
   ```bash
   # 代码检查
   pnpm lint
   pnpm type-check
   
   # 构建测试
   pnpm build
   pnpm test  # 如果有测试
   ```

4. **代码审查**
   - 创建Pull Request
   - 自动化CI/CD检查
   - 人工代码审查
   - 合并到develop分支

5. **部署流程**
   ```bash
   # 准备发布
   git checkout release/v1.0.0
   git merge develop
   
   # 部署到生产环境
   git checkout main
   git merge release/v1.0.0
   git push origin main
   ```

### Vercel部署配置
```json
{
  "framework": "nextjs",
  "buildCommand": "pnpm build",
  "outputDirectory": ".next",
  "installCommand": "pnpm install",
  "devCommand": "pnpm dev",
  "functions": {
    "app/**/*": {

    }
  }
}
```

```bash

NEXT_PUBLIC_ANALYTICS_ID=your-analytics-id

```
